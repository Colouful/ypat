会员闭环一期设计：架构与数据模型

方案 B：规则化会员闭环。套餐卖价格和时长，等级决定权益，权益按业务场景配置。

支付成功后只走一个后端事务：订单置为已支付 → 开通/续期会员 → 发放赠送拍拍豆 → 写收支记录 → 写操作日志。 提交约拍时由后端读取 BASIC 权益规则，计算原价、优惠和实扣。

核心表

member_plan

套餐：名称、时长、售价、划线价、赠送拍拍豆、绑定等级、上下架、排序。

member_level

等级：一期内置 BASIC，保留展示名、状态和说明，未来可扩展 PRO。

member_benefit_rule

权益规则：等级 + 场景 + 优惠类型 + 优惠数量。提交约拍先接入。

user_member

用户会员状态：当前等级、到期时间、来源订单、是否有效。

member_order

会员订单：锁定套餐快照、支付状态、微信流水、赠送豆数量。

member_operation_log

操作日志：后台赠送、延长、取消、支付开通均记录操作者、原因和前后值。
