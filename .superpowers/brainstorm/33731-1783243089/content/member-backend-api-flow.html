会员闭环一期设计：后端接口与支付流

目标：接口鉴权清晰，支付与账务事务一致，提交约拍扣豆由后端统一计算。

接口分组

公开接口

GET /member/plans：套餐列表。允许未登录访问，用于会员页展示。

登录接口

/member/status、/member/order/create、/member/orders、/member/benefit/quote 均需要 Token。

后台接口

/admin/member/plans、/admin/member/rules、/admin/member/users、/admin/member/logs 需要管理员 Token。

支付开通流

微信支付成功后的后端事务
创建待支付订单
微信统一下单
微信回调验签
订单置为已支付
开通会员 + 发豆 + 日志

回调必须幂等：只允许 status=待支付 的订单变为已支付。重复回调返回成功但不重复发放会员和拍拍豆。

提交约拍扣豆流

同一套 BenefitCalculator 供预估与真实提交使用
读取原价
系统参数 5
读取用户会员
BASIC 是否有效
读取权益规则
优惠 2
返回/扣减
实扣 3

预估接口只展示，不扣款；提交接口重新计算并扣款，防止前端篡改优惠金额。

异常与边界

必须处理

  • 套餐下架后不能创建新订单
  • 订单创建后锁定价格和赠送豆
  • 支付成功但回调延迟时前端展示确认中
  • 会员过期时不再享受优惠
  • 后台取消会员后下次预估/提交立即失效

一期不做

  • 自动退款
  • 自动续费
  • 多等级售卖
  • 后台手改订单状态
  • 后台手改拍拍豆余额

当前 401 修复原则

/member/plans 鉴权契约
后端安全白名单加入 GET /member/plans；前端保持 withToken=false。
/member/status 等用户相关接口继续要求 Token，未登录返回 401 并引导登录。
