# 修复 YPAT 后台管理端验收问题清单

## Task 1: P0-001 + P0-002 — 管理员身份校验与角色拦截

这两个问题必须一起修复：任何 t_user 可登录后台 + 普通用户 Token 可访问 /admin/** 接口。

**修复方案**: 在 JWT Token 中嵌入 `admin` 标识，JwtTokenFilter 据此授予 ROLE_ADMIN 权限，WebSecurityConfig 对 /admin/** 要求该角色。同时在 AdminAuthService 增加管理员白名单校验。

### 1.1 修改 `backend/system-wap/src/main/java/com/ypat/comm/Const.java`
- 新增常量 `CLAIM_KEY_ADMIN = "admin"` 和 `ROLE_ADMIN = "ROLE_ADMIN"`
- 新增环境变量 `YPAT_ADMIN_USER_IDS`（逗号分隔的管理员 user ID 白名单）

### 1.2 修改 `backend/system-wap/src/main/java/com/ypat/util/JwtTokenUtil.java`
- 新增 `generateAdminToken(SecurityUserDetails)` 方法，在 claims 中加入 `admin: true`
- 新增 `isAdminToken(String token)` 方法，从 claims 读取 admin 标识

### 1.3 修改 `backend/system-wap/src/main/java/com/ypat/model/SecurityUserDetails.java`
- 新增 `admin` 字段（boolean）
- 修改 `getAuthorities()`: 若 admin=true 返回 `ROLE_ADMIN`，否则返回空列表

### 1.4 修改 `backend/system-wap/src/main/java/com/ypat/service/AdminAuthService.java`
- `login()` 中: 密码验证通过后，校验 user.getId() 是否在管理员白名单中
- Token 生成改用 `jwtTokenUtil.generateAdminToken(userDetails)`
- `getUserInfo()` 中: 增加 admin token 校验

### 1.5 修改 `backend/system-wap/src/main/java/com/ypat/filter/JwtTokenFilter.java`
- 对 `/admin/` 路径的请求: 调用 `jwtTokenUtil.isAdminToken()` 检查是否为管理员 Token
- 若为 admin token，创建 SecurityUserDetails 时设置 admin=true，使 getAuthorities() 包含 ROLE_ADMIN
- 若非 admin token 访问 /admin/ 路径，不设置认证（触发 401）

### 1.6 修改 `backend/system-wap/src/main/java/com/ypat/config/WebSecurityConfig.java`
- 在 `.authorizeRequests()` 中添加: `.antMatchers("/admin/**").hasRole("ADMIN")`
- 放在 `.anyRequest().authenticated()` 之前

### 1.7 新增 `docs/sql/pending/V_admin_role.sql`
- `ALTER TABLE t_user ADD COLUMN is_admin TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否管理员'`
- `UPDATE t_user SET is_admin=1 WHERE id=1`（当前唯一的 Admin 用户）

## Task 2: P1-001 — 产品列表分页计数显示"共0条"

**根因**: `AdminProductController.list()` 使用 `GsonUtils.fromJson(json, Object.class)` 反序列化 Feign 返回的 Spring Data Page JSON。由于 Gson 将数字默认解析为 Double，且 Map 结构可能丢失 totalElements 字段精度。

**修复**: 使用 Gson 的 JsonParser 直接解析为 JsonElement，避免反序列化为 Object 再序列化的精度丢失。

### 修改 `backend/system-wap/src/main/java/com/ypat/controller/AdminProductController.java`
- 将 `GsonUtils.fromJson(json, Object.class)` 替换为 `JsonParser.parseString(json)` 返回 JsonElement
- 同样处理其他使用 `Object.class` 反序列化的 Controller 方法（detail/save/upDown）

### 同步检查并修复其他 Admin Controller
逐一检查 AdminYpatController、AdminUserController、AdminArticleController、AdminBannerController、AdminOrderController、AdminMessController、AdminPubEventController 中相同的 `GsonUtils.fromJson(json, Object.class)` 模式，统一替换为 JsonParser。

## Task 3: P1-002 — 中文编码乱码

**根因**: Spring Boot 1.5.9 不自动配置 HTTP 响应编码为 UTF-8。`application.yml` 中缺少 `spring.http.encoding` 配置。

### 修改 `backend/system-wap/src/main/resources/dev/application.yml`（及 pro/pre 环境）
在 spring 节点下添加:
```yaml
spring:
  http:
    encoding:
      charset: UTF-8
      enabled: true
      force: true
```
