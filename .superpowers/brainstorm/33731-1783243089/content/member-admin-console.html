会员闭环一期设计：管理后台

目标：运营可配置套餐和权益，客服可处理会员状态，财务/运营可查订单与日志；高风险资金字段不允许手工改。

后台菜单结构

建议新增：会员系统
审核系统
查询系统
订单系统
会员系统
套餐管理：新增 / 编辑 / 上下架 / 排序
权益配置：BASIC + 场景 + 优惠数量
会员用户：查询用户会员状态，赠送 / 延长 / 取消
会员订单：支付订单只读查询
操作日志：支付开通和后台人工操作全量留痕

页面职责

套餐管理

配置名称、周期、价格、划线价、赠送拍拍豆、绑定 BASIC、上下架、排序。编辑只影响新订单。

权益配置

一期只启用提交约拍。字段：场景、优惠拍拍豆数量、最低实扣、启用状态、说明。

会员用户

按手机号、昵称、会员状态、到期时间筛选。可赠送、延长、取消会员，必须填写原因。

会员订单

只读查询支付订单、套餐快照、金额、赠送豆、微信流水、支付时间。后台不改支付状态。

操作日志

记录操作者、操作类型、原因、变更前后值、关联订单。支持按用户和时间筛选。

人工调整边界

允许

  • 赠送 BASIC 会员 N 天
  • 延长已有会员 N 天
  • 取消会员资格
  • 查看订单和日志

不允许

  • 手动修改支付订单状态
  • 手动修改微信流水
  • 直接改用户拍拍豆余额
  • 删除操作日志

权限建议

一期沿用现有管理员体系，但操作粒度要留字段
普通管理员：查看套餐、订单、用户会员、日志
运营管理员：编辑套餐、编辑权益规则、上下架套餐
超级管理员：赠送 / 延长 / 取消会员

如果当前后台暂时没有细粒度角色，一期可以先全部挂在管理员下，但后端日志仍保存 operatorId 和 actionType，后续补权限不需要迁移数据。
