# 首页横幅、作品体验、实名认证说明与开屏页设计

日期：2026-07-07
工作区：`/Users/lizhenwei/workspace/vueworkspace/ypat-workspace`
目标端：`91pai-master` 旧小程序、`backend/system-wap` 后端接口层、`frontend-admin` 新版管理后台

## 背景

当前旧小程序首页 `pages/home/home/index` 顶部只有固定 tab(标签切换) 和列表，尚未把管理后台的 banner(横幅) 能力展示到用户首屏。作品能力在后端已经存在 `/work/*` 接口，管理后台也已有作品管理，但旧小程序端还主要停留在旧 `ypat`(约拍) 页面，缺少参考图里的作品详情、点赞、收藏、投诉、拉黑确认、安全提醒和快捷约拍体验。

实名认证当前直接进入表单页 `pages/mine/realname/index`，缺少参考图里的说明页，用户在提交身份证照片和支付审核费前没有充分理解权益。应用启动也没有开屏页，无法承载品牌露出、活动或安全提示。

本次设计目标是在不破坏现有约拍主流程的前提下，补齐这些用户可见能力，并尽量复用现有后端和后台能力。

## 范围

本次包含：

1. 在旧小程序首页 `home-top` 对应的顶部区域下方增加 banner(横幅) 组件，数据来自后端 `/banner/list`，内容由管理后台 banner(横幅) 管理维护。
2. 补齐旧小程序作品页面能力：作品详情、点赞、收藏、分享、投诉、拉黑确认、立即约拍和安全提醒。
3. 如果后端或管理后台缺少作品投诉治理入口，则补齐后台投诉列表、详情和处理能力。
4. 新增实名认证说明页，作为进入现有实名认证表单页前的引导页。
5. 在旧小程序前端增加约 3 秒开屏页，可跳过。
6. 补充关键单元测试、source test(源码测试) 或 smoke test(冒烟测试)，锁住接口、路由和基础交互。

本次不包含：

1. 不重构旧 `ypat`(约拍) 数据模型。
2. 不迁移历史约拍详情到作品详情。
3. 不重做 tabBar(底部导航) 或全站视觉系统。
4. 不改造新版小程序 `frontend`，除非实施时发现旧小程序能力依赖新版共享接口文档。
5. 不把开屏页接入远程广告投放系统；本次只支持本地默认内容和未来可接 banner(横幅) 数据的扩展点。

## 现状判断

### 已存在能力

- 后端 `BannerController(横幅控制器)` 已提供 `/banner/list`，并过滤已发布状态。
- 管理后台 `frontend-admin/src/views/banner/list` 已有 banner(横幅) 列表、编辑、发布和撤回能力。
- 后端 `WorkController(作品控制器)` 已提供 `/work/get`、`/work/list`、`/work/like/add`、`/work/sc/add`、`/work/quick-apply` 等作品接口。
- 后端 `WorkComplainController(作品投诉控制器)` 已提供 `/work/complain`，并带 60 秒 Redis(缓存数据库) 限频。
- 管理后台 `frontend-admin/src/views/manage/work-list` 已有作品列表、详情、审核和下架能力。
- 实名认证表单页 `pages/mine/realname/index`、OCR(光学字符识别) 和提交接口已存在。

### 需要补齐能力

- 旧小程序没有 `/banner/list` 的接口包装和首页 banner(横幅) 组件。
- 旧小程序没有独立 `work`(作品) 页面目录和接口包装。
- 作品投诉前端页、证据截图上传、拉黑确认弹窗缺失。
- 后台是否已有投诉治理页不明确；若没有，需要补 `AdminWorkComplainController(后台作品投诉控制器)` 和前端页面。
- 实名认证缺少说明页和入口跳转策略。
- App(应用) 启动缺少开屏遮罩或开屏页面。

## 方案选择

采用方案 A：补齐新 `work`(作品) 体系，旧 `ypat`(约拍) 保持兼容。

理由：

1. 后端作品接口已经相对完整，复用成本低。
2. 作品详情和约拍详情语义不同，拆开后更容易维护。
3. 投诉、点赞、收藏、快捷约拍都属于作品治理和互动能力，不适合继续塞入旧约拍详情页。
4. 首页 banner(横幅)、实名认证说明页和开屏页都可以独立落地，不需要大规模业务重构。

备选方案不采用：

- 直接把 `pages/home/desc/index` 改成作品详情：短期快，但会让约拍详情和作品详情继续混杂。
- 只做前端静态页面：无法满足投诉、拉黑、后台治理等闭环。

## 用户体验设计

### 首页 banner(横幅)

位置：

- 放在固定 tab(标签切换) 下方、消息提示和首页列表上方。
- 由于当前 `.tab-box` 是 fixed(固定定位)，banner(横幅) 放在 `.content-box` 内首位，避免遮挡。

视觉：

- 卡片宽度左右留白 24rpx。
- 圆角 18rpx，使用 `aspectFill(裁剪填充)`。
- 底部加轻微渐变遮罩展示标题。
- 右下角显示分页点。
- banner(横幅) 少于 2 张时不自动轮播。

交互：

- 点击 banner(横幅) 时，如果后端返回跳转字段则跳转；当前 `BannerQo(横幅请求对象)` 已知字段有限，第一期点击只预览图片或进入 `/banner/get` 详情。
- 接口失败或无数据时不展示组件，不影响首页列表。

### 作品详情页

参考图 #1，页面标题为“作品详情”。

内容：

- 原因必选。

### 开屏页

实现方式：

- 采用根层 `SplashOverlay(开屏遮罩)`，挂在 `App.vue` 或首页根组件中。
- 不改 tabBar(底部导航) 首页路径，降低启动链路风险。

行为：

- 展示约 3 秒。
- 右上角“跳过”按钮。
- 倒计时结束自动关闭。
- 关闭后记录 `splash_last_show_date`，默认同一自然日只展示一次。

视觉：

- 使用品牌 logo(标志) 与粉红到珊瑚色渐变。
- 中央展示“爱去拍”与安全/约拍文案。
- 底部展示“3s 后进入”倒计时。

## 数据流

### 首页 banner(横幅)

1. 首页 onShow(页面显示) 或 onLoad(页面加载) 调用 `banner_list`。
2. `banner_list` 请求 `/banner/list?page=0&size=5`。
3. `homeBanner` 接收数组，过滤没有 `imgpath` 的数据。
4. 用户点击 banner(横幅) 后执行预览或跳转。

### 作品详情

1. 页面接收 `id`。
2. 调用 `/work/get?id=...`。
3. 渲染作者、媒体、计数和状态。
4. 点赞/收藏调用对应接口，成功后本地乐观更新；失败则回滚。

### 投诉

1. 页面接收 `workId` 和被投诉用户信息。
2. 用户填写原因、内容和证据。
3. 证据图如后端 `WorkComplainQo(作品投诉请求对象)` 支持 URL(链接)，则先走上传；如支持 base64(图片编码)，则直接提交压缩后的 base64(图片编码)。
4. 调用 `/work/complain`。
5. 后端限频和鉴权失败由统一错误提示处理。

### 快捷约拍

1. 页面接收 `workId`。
2. 首次展示安全提醒。
3. 用户填写约拍理由和联系方式。
4. 调用 `/work/quick-apply`。
5. 成功进入现有成功页或展示提交成功 toast(轻提示)。

## 接口契约

### 小程序新增 API(接口) 包装

新增方法：

- `banner_list(data)` -> `GET /banner/list`
- `work_get(data)` -> `GET /work/get`
- `work_like_add(data)` -> `PUT /work/like/add`
- `work_like_cancel(data)` -> `PUT /work/like/cancel`
- `work_sc_add(data)` -> `PUT /work/sc/add`
- `work_sc_cancel(data)` -> `PUT /work/sc/cancel`
- `work_quick_apply(data)` -> `POST /work/quick-apply`
- `work_complain(data)` -> `POST /work/complain`

### 后台投诉治理接口

若实施时确认后端不存在后台投诉治理，新增：

- `GET /admin/work/complain/list`
  - 参数：`page`、`size`、`status`、`workId`、`userId`
  - 返回：分页投诉列表。
- `GET /admin/work/complain/detail`
  - 参数：`id`
  - 返回：投诉详情、证据截图、作品摘要、投诉人和被投诉人。
- `POST /admin/work/complain/handle`
  - 参数：`id`、`status`、`reason`、`offlineWork`
  - 返回：处理结果。

## 错误处理

- banner(横幅) 加载失败：静默隐藏，不阻断首页。
- 作品详情加载失败：显示空状态和重试按钮。
- 点赞/收藏失败：恢复原状态并提示。
- 未登录：跳转 `/pages/login/login/index`。
- 投诉限频：展示后端返回的频控提示。
- 投诉证据上传失败：提示用户重新上传，不提交半成品。
- 快捷约拍失败：展示后端 message(消息)，不扣减本地展示的拍拍豆。
- 开屏倒计时异常：默认 3 秒后关闭，避免卡住首页。

## 安全与隐私

- 投诉内容和证据截图必须走登录态。
- 投诉证据不在前端日志打印。
- 实名认证说明页不展示身份证示例之外的敏感信息。
- 开屏页不读取用户隐私信息。
- 如果新增后台投诉治理接口，必须走管理端鉴权，不能开放给匿名用户。

## 开放问题与默认决策

- 作品页面默认使用新 `work`(作品) 体系，不改造旧 `ypat`(约拍) 详情页。
- banner(横幅) 第一期只使用现有 `title`、`imgpath` 和状态字段；跳转字段若后端对象已有但前端未声明，实施时再接入。
- 拉黑功能若后端无接口，第一期只展示确认弹窗并保留接口扩展点，不假装已拉黑成功。
- 开屏页同一自然日只展示一次，避免用户每次切回应用都被打扰。
