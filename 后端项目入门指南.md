# YPAT（爱去拍）后端项目入门指南

> 写给第一次接触后端开发的同学，从零开始了解这个项目的方方面面。

---

## 一、这个项目是做什么的？

### 1.1 一句话介绍
YPAT（爱去拍）是一个**摄影约拍平台**的后端服务，类似于"摄影界的滴滴"——摄影师和模特在平台上互相找到对方，发起约拍，完成交易。

### 1.2 核心业务流程

```
用户注册 → 浏览约拍 → 发起申请 → 支付拍拍豆 → 约拍成功
   ↑                                              ↓
   └──────── 微信消息通知 ← 审核通过 ← 发布约拍 ←──┘
```

### 1.3 用户角色

| 角色 | 说明 | 典型行为 |
|------|------|----------|
| 摄影师 | 平台核心用户 | 发布约拍、查看申请、收钱拍照 |
| 模特 | 平台核心用户 | 浏览约拍、发起申请、付费约拍 |
| 妆造师 | 服务提供者 | 发布妆造服务 |
| 修图师 | 服务提供者 | 发布修图服务 |
| 商家 | 影楼/工作室 | 批量发布约拍 |
| 管理员 | 平台运营 | 审核约拍、审核实名、管理内容 |

## 二、项目包含哪些功能？

### 2.1 用户模块

#### 注册登录
- **微信小程序登录**: 用户打开小程序 → 微信授权 → 获取code → 换取用户信息
- **百度小程序登录**: 同上，支持百度小程序
- **JWT Token认证**: 登录后生成Token，后续请求携带Token验证身份

#### 用户资料
- 基本信息：昵称、性别、职业、手机号、微信、QQ、生日、地区
- 头像上传：支持拍照和从相册选择，存储到文件服务器
- 职业类型：摄影师、模特、妆造师、修图师、个人、演员、商家、其他

#### 实名认证
- 身份证OCR：调用百度AI识别身份证信息（姓名、身份证号）
- 证件照上传：身份证正反面照片
- 人工审核：管理员审核认证申请
- 认证状态：暂存 → 待审核 → 通过/不通过

### 2.2 约拍模块

#### 发布约拍
用户填写约拍信息：
- **描述**: 约拍主题、要求
- **目标**: 想找什么样的人
- **拍摄日期**: 什么时候拍
- **拍摄地区**: 在哪拍（省市区）
- **拍摄风格**: 复古/INS/胶片/少女/暗黑/情绪/夜景/欧美/商务/韩系/日系/情侣/样片
- **收费方式**: 希望互勉/我要收费/可付费/费用协商
- **收费金额**: 具体多少钱
- **图片**: 作品展示图（自动加水印防盗）

#### 约拍审核
- 用户提交后状态为"待审核"
- 管理员审核通过后才展示给其他用户
- 审核不通过会通知用户原因

#### 约拍列表
- **推荐列表**: 管理员标记的优质约拍
- **最新列表**: 按发布时间排序
- **筛选条件**: 地区、风格、收费方式等

#### 申请约拍
- 用户看到感兴趣的约拍 → 发起申请
- 扣除拍拍豆（虚拟货币）
- 通知发布者

#### 收藏功能
- 收藏感兴趣的约拍
- 查看收藏列表

### 2.3 消息模块

#### 消息类型
| 类型 | 说明 | 触发场景 |
|------|------|----------|
| 约拍消息 | 约拍申请通知 | 有人向你发起约拍 |
| 实名认证审核 | 认证结果通知 | 管理员审核实名认证 |
| 发布信息审核 | 约拍审核结果 | 管理员审核约拍发布 |
| 已查看消息 | 对方已读通知 | 对方查看了你的消息 |
| 订单通知 | 支付成功通知 | 拍拍豆充值成功 |

#### 消息推送
- **微信模板消息**: 重要消息通过微信推送
- **站内消息**: 平台内消息列表

### 2.4 支付模块

#### 拍拍豆（虚拟货币）
- 平台统一使用"拍拍豆"作为虚拟货币
- 消耗场景：
  - 发布约拍：消耗3个
  - 申请约拍：消耗1个
  - 查看联系方式：消耗5个
- 充值方式：微信支付

#### 订单流程
```
选择充值金额 → 创建订单 → 调用微信支付 → 用户付款 
→ 微信回调通知 → 更新订单状态 → 拍拍豆到账
```

#### 账单记录
- 记录每笔支付的详细信息
- 支持查询和导出

### 2.5 内容模块

#### 文章管理
- 文章发布：标题、内容、封面图
- 状态管理：暂存、发布、撤回
- 热门标记：标记优质文章

#### Banner管理
- 首页轮播图
- 状态管理：发布、撤回

### 2.6 文件上传

#### 图片存储
- 使用FastDFS分布式文件存储
- 支持多图上传
- 自动添加水印（约拍图片）

### 3.2 模块说明

#### 基础设施层（base-service）
| 模块 | 作用 | 类比 |
|------|------|------|

| system-restapi | REST API | 其他系统调用的接口 |
| system-sso | 单点登录 | 统一登录认证 |

## 五、部署架构详解

### 5.1 需要部署几个项目？

**总共需要部署 10 个服务**，分为四类：

#### 第一类：基础中间件（3个）
| 序号 | 服务 | 端口 | 作用 | 必须？ |
|------|------|------|------|--------|
| 1 | MySQL | 3306 | 数据库，存储所有业务数据 | ✅ 必须 |
| 2 | Redis | 6379 | 缓存，存储Token、Session | ✅ 必须 |
| 3 | FastDFS Tracker | 22122 | 文件存储-跟踪服务 | ✅ 必须 |
|   | FastDFS Storage | 23000 | 文件存储-存储服务 | ✅ 必须 |

**FastDFS 说明：**
- 用于存储用户头像、约拍图片、身份证照片、文章封面等
- 由两部分组成：Tracker（调度）+ Storage（存储）
- 访问端口：通过 Nginx 代理，端口 8888

#### 第二类：基础设施服务（2个）
| 序号 | 服务 | 端口 | 作用 | 必须？ |
|------|------|------|------|--------|
| 3 | base-eureka | 8761 | 注册中心，服务发现 | ✅ 必须 |
| 4 | base-config | 8888 | 配置中心，统一配置管理 | ✅ 必须 |

#### 第三类：业务服务（4个）
| 序号 | 服务 | 端口 | 作用 | 必须？ |
|------|------|------|------|--------|
| 5 | system-sso | 8082 | 单点登录，用户认证 | ✅ 必须 |
| 6 | system-wap | 8080 | 移动端API，小程序调用 | ✅ 必须 |
| 7 | system-web | 8081 | 管理后台，运营使用 | ✅ 必须 |
| 8 | system-restapi | 8083 | REST API，其他系统调用 | ⚠️ 可选 |

## 六、如何部署到服务器？

### 5.1 部署方式对比

| 方式 | 优点 | 缺点 | 适合场景 |
|------|------|------|----------|
| 传统部署 | 简单直接 | 配置繁琐 | 开发测试 |
| Docker部署 | 环境隔离 | 需学Docker | 生产环境 |
| 云平台部署 | 省心省力 | 费用较高 | 不想运维 |

### 5.2 传统部署步骤

#### 第一步：安装基础软件
```bash
# 安装JDK 8
yum install -y java-1.8.0-openjdk java-1.8.0-openjdk-devel

# 安装Maven
yum install -y maven

# 安装MySQL 5.7
yum install -y mysql-server
systemctl start mysqld
systemctl enable mysqld

# 安装Redis
yum install -y redis
systemctl start redis
systemctl enable redis

# 安装Nginx
yum install -y nginx
systemctl start nginx
systemctl enable nginx
```

#### 第二步：创建数据库
```bash
# 登录MySQL
mysql -u root -p

# 创建数据库
CREATE DATABASE ypat DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 创建用户（可选）
CREATE USER 'ypat'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON ypat.* TO 'ypat'@'localhost';
FLUSH PRIVILEGES;
```

#### 第三步：修改配置文件
```bash
# 编辑配置中心配置
vi backend-base/base-config/src/main/resources/application.yml

# 主要配置项：
# - 数据库连接地址、用户名、密码
# - Redis连接地址、密码
# - 微信小程序AppID、AppSecret
# - 微信支付商户号、密钥
# - 百度AI API Key、Secret Key
```

#### 第四步：打包项目
```bash
# 进入后端目录
cd backend

# 打包（跳过测试）
mvn clean package -DskipTests

# 打包完成后，每个模块的target目录下会有jar包
```

#### 第五步：启动服务
```bash
# 1. 启动注册中心
nohup java -jar backend-base/base-eureka/target/base-eureka-1.0-SNAPSHOT.jar &

# 等待30秒，让注册中心完全启动

# 2. 启动配置中心
nohup java -jar backend-base/base-config/target/base-config-1.0-SNAPSHOT.jar &

# 等待30秒

# 3. 启动业务服务
nohup java -jar backend/system-sso/target/system-sso-1.0-SNAPSHOT.jar &
nohup java -jar backend/system-wap/target/system-wap-1.0-SNAPSHOT.jar &
nohup java -jar backend/system-web/target/system-web-1.0-SNAPSHOT.jar &
```

#### 第六步：配置Nginx
```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 移动端API
    location /api/ {
        proxy_pass http://127.0.0.1:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 管理后台
    location /admin/ {
        proxy_pass http://127.0.0.1:8081/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 5.3 Docker部署步骤

#### 第一步：安装Docker
```bash
# 安装Docker
curl -fsSL https://get.docker.com | sh
systemctl start docker
systemctl enable docker

# 安装Docker Compose
curl -L "https://github.com/docker/compose/releases/download/v2.20.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
chmod +x /usr/local/bin/docker-compose
```

#### 第二步：使用docker-compose.yml
项目根目录已有`docker-compose.yml`，直接运行：
```bash
# 构建并启动所有服务
docker-compose up -d

# 查看运行状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

## 九、常见问题解答

### Q1: 这个项目难吗？新手能看懂吗？
A: 项目本身不复杂，主要是CRUD操作。难点在于Spring Cloud微服务的概念，建议先学Spring Boot基础。

### Q2: 需要学多久才能独立开发？
A: 如果有Java基础，大概需要：
- Spring Boot基础：1-2周
- Spring Cloud概念：1-2周
- 项目熟悉：1周
- 总计：1个月左右

### Q3: 可以用其他语言重写吗？
A: 可以。后端只是提供API接口，用Python/Node.js/Go都可以重写，只要接口格式保持一致。

### Q4: 数据库可以换成其他数据库吗？
A: 可以。项目使用JPA作为ORM，切换数据库只需要改配置，不需要改代码。

### Q5: 必须用微服务架构吗？
A: 不必须。如果只是学习或小项目，可以合并成单体应用，去掉Eureka、Config等基础设施。

### Q6: 微信支付必须有营业执照吗？
A: 是的。微信支付需要企业资质，个人无法申请。开发阶段可以用沙箱环境测试。

### Q7: 服务器费用大概多少？
A: 开发测试：
- 阿里云/腾讯云学生机：9.9元/月
- 普通云服务器：50-100元/月

生产环境：
- 2核4G：100-200元/月
- 4核8G：300-500元/月

### Q8: 代码看不懂怎么办？
A: 建议阅读顺序：
1. 先看entity（实体类）了解数据结构
2. 再看controller（控制器）了解接口
3. 最后看service（服务）了解业务逻辑
