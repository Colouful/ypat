# 后台内测数据资源选择弹窗设计

**日期**：2026-07-11

**状态**：已确认

**适用范围**：新版后台“内测数据生成”页面

## 背景

当前“新增作品”通过下拉框选择作品组，“新增约拍”则没有资源选择入口，而是由后端自动挑选一张约拍图片。该交互不利于浏览、筛选和确认资源，也无法让一条约拍携带多张图片。

本设计将作品组和约拍图片的选择统一改为弹窗表格。主表单中的多选风格只描述最终发布内容，不参与弹窗资源过滤；弹窗使用自己的筛选条件。

## 目标

- 使用弹窗完成作品组和约拍图片选择。
- 弹窗顶部提供 Tab(标签页)，支持图片和视频资源浏览。
- 作品一次只能选择一个作品组，该组全部资源生成一条作品。
- 约拍最多选择 9 张图片，全部图片归属同一条约拍。
- 弹窗支持分页、独立筛选和跨页保留选择。
- 提交前明确展示已选资源摘要。
- 后端重新校验资源状态，并通过事务保证生成和资源占用的一致性。

## 非目标

- 不支持约拍视频发布。约拍弹窗的视频页只允许浏览。
- 不允许一次选择多个作品组，也不合并不同作品组。
- 不根据主表单已选风格自动筛选弹窗资源。
- 不修改内测资源的入库、编辑和分组规则。
- 不新增数据库表或约拍视频字段。

## 方案选择

采用共享 `ResourcePickerDialog`(资源选择弹窗组件)。组件复用弹窗框架、标签页、筛选、分页和临时选择状态，但作品组表格与约拍资源表格分别渲染，避免强行统一不同的数据结构。

未采用以下方案：

- 将两套弹窗直接写入生成页面：会继续扩大已经较长的页面文件。
- 创建两个完全独立的弹窗：会重复标签页、筛选、分页和选择状态逻辑。

## 页面结构

### 生成页面

生成页面保留现有主表单，并替换原作品媒体和作品组下拉框：

- 新增作品显示“选择作品组”按钮、已选组标题、媒体数量和“清空”操作。
- 新增约拍显示“选择约拍图片”按钮、已选图片数量、缩略图摘要和“清空”操作。
- 主表单风格继续支持最多 5 项多选，只写入作品标签或约拍风格。
- 提交前要求作品已选一个作品组，约拍已选 1 至 9 张图片。

### 资源选择弹窗

弹窗建议宽度为 `960px`，从上到下包含：

1. 图片、视频 Tab(标签页)。
2. 当前业务模式对应的筛选栏。
3. 带选择列的资源表格。
4. 分页、已选数量、取消和确认操作。

弹窗内部维护临时选择。点击“确认选择”后才回传生成页面；点击取消、关闭或遮罩区域不改变生成页面原有选择。

### 作品组详情

作品组表格操作列提供“详情”按钮。详情使用独立轻量弹窗，以网格展示组内资源：

- 图片展示固定比例缩略图、标题和资源地址。
- 视频展示媒体类型、标题和资源地址，不自动播放。
- 详情弹窗不改变作品组选择。

## 作品选择设计

### 查询

弹窗调用：

```text
GET /admin/internal-test/resource-groups
```

固定查询条件：

- `usageType=work`(用途为作品)
- `mediaType`(媒体类型)取当前图片或视频标签

弹窗筛选条件：

- `keyword`(关键词)
- `styleCode`(资源风格)
- `usedFlag`(占用状态)
- `page`(页码)
- `size`(每页数量)

筛选条件不读取主表单的 `styleCodes`(发布风格列表)。

### 选择

- 表格第一列使用复选框。
- 业务上只允许保留一个作品组。
- 勾选新组时取消旧组。
- 切换标签页、翻页和修改筛选条件时保留当前已选组。
- 点击重置只清空筛选条件，不清空已选组。

### 提交

前端继续使用兼容字段 `groupNos`(作品组编号列表)，但数组长度必须为 1：

```json
{
  "groupNos": ["ITG202607110001"]
}
```

后端必须再次校验数组长度为 1，防止绕过前端一次生成多个作品。选中组内全部资源写入同一条作品的媒体记录。

## 约拍选择设计

### 查询

弹窗调用：

```text
GET /admin/internal-test/resources
```

固定查询条件：

- `usageType=ypat`(用途为约拍)
- `mediaType`(媒体类型)取当前图片或视频标签

弹窗筛选条件：

- `keyword`(关键词)
- `styleCode`(资源风格)
- 省、市、区三级地区
- `usedFlag`(占用状态)
- `page`(页码)
- `size`(每页数量)

筛选条件不读取主表单的 `styleCodes`(约拍风格列表)。

### 选择

- 图片页允许勾选 1 至 9 张资源。
- 达到 9 张后，其他未选行禁用；取消已选图片后可继续选择。
- 视频页展示查询结果，但选择框全部禁用，并提示“当前约拍仅支持图片资源”。
- 选择使用资源编号作为稳定标识，通过映射保存跨页资源摘要。
- 切换标签页、翻页和修改筛选条件时保留已选图片。
- 点击重置只清空筛选条件，不清空已选图片。

### 提交

确认选择后，将资源编号按选择顺序写入 `ypatResourceIds`(约拍资源编号列表)：

```json
{
  "ypatResourceIds": [101, 108, 115]
}
```

后端不再为该入口自动挑选资源。未传资源、传入空列表或超过 9 项时返回明确参数错误。

## 后端生成流程

### 作品

1. 校验用户为内测用户。
2. 校验 `groupNos`(作品组编号列表)长度为 1。
3. 加载作品组全部资源。
4. 校验资源均启用、未占用、用途正确且组内媒体类型一致。
5. 创建一条作品并写入组内全部媒体。
6. 标记组内全部资源占用。

### 约拍

1. 校验主表单字段和内测用户。
2. 校验 `ypatResourceIds`(约拍资源编号列表)数量为 1 至 9，且不存在重复编号。
3. 一次加载全部资源，并校验每项均满足：
   - 资源存在。
   - 状态为启用。
   - 尚未占用。
   - 用途为约拍。
   - 媒体类型为图片。
4. 更新所选内测用户的微信号和联系电话。
5. 创建一条约拍。
6. 按前端选择顺序写入多条 `t_ypat_img`(约拍图片表)记录。
7. 将全部图片资源标记为同一批次、同一约拍占用。

生成过程沿用事务。任一校验、保存或占用失败时，用户联系方式、约拍、约拍图片和资源占用全部回滚。

## 并发与状态一致性

弹窗展示的资源状态可能在提交前发生变化，因此前端选择不能替代后端校验。

- 后端提交时重新读取并校验全部资源。
- 资源占用沿用现有原子更新能力。
- 任一资源已被其他操作占用时，整个生成请求失败，不生成部分约拍图片。
- 前端保留主表单和已选摘要，提示用户重新打开弹窗刷新选择。

## 错误处理

前端提交前提示：

- 新增作品未选择作品组：`请选择作品组`。
- 新增约拍未选择图片：`请选择约拍图片`。
- 约拍图片超过 9 张：`约拍图片最多选择9张`。

后端错误应保留具体原因，例如：

- `一次只能选择一个作品组`。
- `约拍图片不能为空`。
- `约拍图片最多选择9张`。
- `约拍资源包含视频`。
- `约拍资源已被占用，请重新选择`。
- `约拍资源用途错误`。

## 组件状态约定

`ResourcePickerDialog`(资源选择弹窗组件)接收：

- 业务模式：作品或约拍。
- 弹窗是否显示。
- 当前已选作品组或约拍图片。

组件对外发出：

- 确认后的选择结果。
- 关闭事件。

组件内部状态包括：

- 当前媒体标签。
- 独立筛选条件。
- 当前页数据和分页总数。
- 临时选择映射。
- 查询和确认加载状态。

组件每次打开时根据外部已选值初始化临时选择；取消时直接丢弃临时变化。

## 测试策略

### 前端回归测试

- 作品模式勾选新组后只保留一个组。
- 约拍模式最多选择 9 张图片。
- 约拍视频页不可选择。
- 翻页、切换筛选和标签后保留已选项。
- 取消不修改外部选择，确认才回填。
- 主表单风格变化不触发资源查询。
- 作品请求只提交一个组编号。
- 约拍请求提交全部已选图片编号。

### 后端回归测试

- 多张约拍图片只生成一条约拍和多条图片记录。
- 多个作品组请求被拒绝。
- 禁用、已占用、非约拍用途或视频资源不能用于约拍。
- 重复资源编号被拒绝。
- 任一资源失效时不生成约拍、不更新联系方式、不占用资源。
- 成功后全部图片资源归属同一批次和同一约拍。

### 手工验收

- 图片与视频标签切换。
- 两种业务模式的独立筛选和分页。
- 跨页选择状态。
- 作品组详情。
- 作品组单选替换。
- 约拍图片 9 张上限。
- 生成后的作品媒体、约拍图片和资源占用状态。

项目实施阶段不下载依赖、不执行构建测试，只增加针对改动的测试代码并执行静态格式检查，运行态由开发者手工验收。

## 验收标准

1. 新增作品不再使用下拉框选择资源。
2. 作品弹窗按作品组分页展示，支持查看组内资源详情。
3. 一次只能选择一个作品组，且只生成一条作品。
4. 新增约拍必须手动选择 1 至 9 张图片。
5. 多张约拍图片只生成一条约拍。
6. 约拍视频资源可浏览但不可选择。
7. 弹窗筛选条件与主表单风格互不影响。
8. 翻页和筛选后已选项仍保留。
9. 取消弹窗不改变原选择。
10. 提交时后端重新校验全部资源，并保证事务一致性。
