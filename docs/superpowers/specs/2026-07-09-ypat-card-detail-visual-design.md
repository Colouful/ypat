# 新版小程序约拍卡片与详情页视觉优化设计

## 背景

新版小程序首页 `pages/home/index`（首页页面）和 `pages-sub/ypat/detail`（约拍详情页面）需要强化约拍信息的可信度展示，并优化当前卡片和详情布局。用户关注三类状态：实名认证、担保金、会员身份。当前前端已经有 `realnameflag`（实名标记）和 `creditflag`（信用担保标记），但会员状态没有随约拍作者信息返回，因此需要后端补充真实会员字段。

## 目标

- 首页隐藏 `home-message`（消息入口）元素，不展示入口。
- 首页 `chip-row`（筛选标签行）顺序调整为：全部、约模特、约摄影师、希望互勉、可付费、INS、胶片、情绪、灵感发现。
- 首页 `KeepYpatCard`（约拍卡片组件）重新优化视觉层级，让图片、标题、目标、费用、作者、城市、互动数据更清晰。
- 首页卡片展示认证状态和担保金状态，包括已认证/未认证、已缴担保金/未缴担保金。
- 如果作者是会员，在作者名称旁展示会员小标识。
- 详情页 `YpatDetailView`（约拍详情视图组件）优化布局，在主信息区和作者卡片中展示认证、担保金、会员状态。
- 保留原有报名、收藏、主页、分享、图片预览、筛选和加载逻辑。

## 非目标

- 不改旧版前端或 `91pai-master`（旧版前端目录）。
- 不重做首页整体信息架构，不新增新的业务入口。
- 不改变约拍报名、收藏、阅读量累加、筛选接口语义。
- 不引入新的 UI（用户界面）组件库。

## 后端设计

在 `backend/system-object/src/main/java/com/ypat/UserQo.java` 的 `UserQo`（用户传输对象）中新增字段：

- `Boolean memberActive`（会员是否生效）
- `String memberLevel`（会员等级）

在 `backend/system-domain/src/main/java/com/ypat/service/YpatInfoService.java` 的约拍列表和详情组装逻辑中，为作者 `userQo`（用户信息对象）补充会员状态。

有效会员判断规则：

- `t_user_member`（用户会员表）存在对应作者记录。
- `level`（等级）不为空且不等于 `NONE`。
- `expireAt`（过期时间）晚于当前时间。

当会员有效时：

- `memberActive = true`
- `memberLevel = level`

当会员无效或查询不到会员记录时：

- `memberActive = false`
- `memberLevel` 可为空。

后端只在返回约拍列表和约拍详情时补充作者会员字段，不改变会员购买、续费、过期规则。

## 前端数据设计

在新版前端 `frontend/src/api/types/index.ts` 的 `UserInfo`（用户信息类型）中新增：

- `memberActive?: boolean`
- `memberLevel?: string`

在首页 `frontend/src/pages/home/index.vue` 的 `cardItems`（卡片数据映射）中增加：

- `memberActive: item.userQo?.memberActive === true`
- `memberLevel: item.userQo?.memberLevel`

在 `KeepYpatCardItem`（约拍卡片数据类型）中增加同名字段，作为卡片渲染来源。

详情页直接从 `detail.userQo.memberActive` 和 `detail.userQo.memberLevel` 读取作者会员状态。

## 首页视觉设计

### 顶部入口

`home-message`（消息入口）从模板中隐藏或移除展示，不再占据顶部右侧空间。搜索框可自然扩展，使顶部更干净。消息跳转方法和未读数刷新逻辑可以暂时保留，减少无关改动。

### 筛选标签

`quickChips`（快速筛选标签）顺序调整为：

1. 全部
2. 约模特
3. 约摄影师
4. 希望互勉
5. 可付费
6. INS
7. 胶片
8. 情绪
9. 灵感发现

对应点击行为保持不变。

### 约拍卡片

`KeepYpatCard`（约拍卡片组件）采用更清晰的三层结构：

- 顶层：封面图 + 需求标题，标题保留两行截断。
- 中层：目标标签、费用标签、认证标签、担保金标签。
- 底层：头像、作者名称、会员小标识、城市、约拍数和时间。

认证标签文案：

- 已认证
- 未认证

担保金标签文案：

- 已缴担保金
- 未缴担保金

会员小标识展示在昵称旁，建议文案为 `VIP` 或 `会员`。样式使用金色系小胶囊，面积小于认证/担保标签，避免喧宾夺主。

样式方向：

- 卡片继续使用现有圆角、阴影和留白体系。
- 状态标签使用不同但克制的颜色区分：认证偏蓝绿，担保偏金色，未完成状态使用浅灰。
- 文本不使用过大的字号，优先提升信息密度和可扫读性。

## 详情页视觉设计

`pages-sub/ypat/detail.vue` 只是承载 `YpatDetailView`（约拍详情视图组件），主要改动在 `frontend/src/components/business/YpatDetailView.vue`。

详情页保留大图轮播，但内容区优化为：

- 核心标签行：约拍目标、费用方式、城市。
- 信任状态行：认证状态、担保金状态、会员身份。
- 标题和正文：标题更明确，正文保持舒适行高。
- 数据行：浏览、约拍、收藏。
- 作者卡片：头像、昵称、会员小标识、认证/担保标签、简介、查看主页按钮。
- 作品区：保留现有图片网格。

作者描述不再只用一句 `摄影爱好者 · 已实名 · 信用担保` 承载关键信任信息，关键信息改为独立标签展示。

## 错误与兼容

- 如果后端没有返回 `userQo`（用户信息对象），前端按匿名用户处理。
- 如果 `memberActive`（会员是否生效）缺失，按非会员展示。
- 如果 `realnameflag`（实名标记）缺失，按未认证展示。
- 如果 `creditflag`（信用担保标记）缺失，按未缴担保金展示。
- 图片为空时继续使用现有默认封面和默认头像。

## 测试方案

前端：

- 更新 `KeepYpatCard`（约拍卡片组件）相关单元测试，覆盖已认证、未认证、已缴担保金、未缴担保金、会员标识。
- 运行 `pnpm run type-check`（类型检查）。
- 运行相关 `vitest`（单元测试）或全量 `pnpm run test`（测试）。

后端：

- 为 `YpatInfoService`（约拍信息服务）的作者会员状态补充逻辑增加或更新单元测试。
- 覆盖有效会员、过期会员、无会员记录三种情况。

验收：

- 首页不展示消息入口。
- 首页筛选标签顺序符合确认顺序。
- 首页每张卡片都能明确看到认证和担保金状态。
- 会员作者昵称旁出现会员标识，非会员不出现。
- 详情页主信息区和作者卡片都能明确看到认证、担保金、会员状态。
