# 爱去拍 Keep 风格 UI 重构设计

## 背景

本次重构以 `/Users/lizhenwei/Downloads/001yp/aiqupai-ui-keep.html` 和用户提供的 4 张参考图为唯一视觉基准，将爱去拍前端迁移为 Keep 风格：清爽白底、浅灰页面背景、Keep 绿主色、线性图标、圆角白卡、胶囊筛选、底部固定操作区。

项目当前前端为 `uni-app` + `Vue 3`，已有 `frontend/src/styles/tokens.scss`、`mixins.scss`、页面路由和业务接口。重构范围限定在前端页面和视觉组件，不改后端、不改接口契约、不改支付与登录安全逻辑。

## 目标

1. `pages/home/index`、`pages/publish/index`、`pages/mine/index` 三个核心页面按截图和 HTML 做 1:1 视觉还原，优先保证布局顺序、间距、圆角、卡片比例、绿色选中态、底部区域和层级关系。
2. `pages-sub/ypat/detail`、`pages/login/index` 按 HTML 中已有详情页和登录页风格还原。
3. 其余页面按同一套 Keep 风格视觉系统延展，保证整体观感一致。
4. 保留现有真实业务能力，包括列表加载、分页、下拉刷新、登录判断、发布提交、图片选择、拍拍豆余额校验、详情报名和收藏。
5. 在新 `worktree` 分支中实施，保护当前已有未提交改动。

## 设计方案

采用“设计系统先行 + 页面批量迁移”的方案。

先把 HTML 里的视觉语言沉淀到 `styles` 和轻量公共组件中，再迁移页面。这样可以避免 25 个页面重复写样式，也便于后续统一调整主色、圆角、阴影、卡片密度和按钮状态。

不采用逐页复制样式的方案，因为它会带来大量重复和维护风险。不采用先做纯静态壳再回接数据的方案，因为当前项目已有业务流程，重构应保留真实数据与交互。

## 视觉系统

继续使用 `frontend/src/styles/tokens.scss`，补齐或校准以下语义：

- 深绿：`#17A857`

- 文字主色：`#1A1D1F`
- 次级文字：`#83888F`
- 弱提示文字：`#B3B8BE`
- 页面背景：`#F5F6F8`
- 卡片背景：`#FFFFFF`
- 灰色胶囊：`#F1F3F5`
- 边线：`#EEF0F2`
- 橙色强调：`#FF9F1C`
- 红色提醒：`#FF4D4F`

### 通用模式

在 `mixins.scss` 和公共组件中沉淀：

- 页面容器：浅灰背景、底部安全区预留。
- 白色卡片：16px 左右圆角、轻阴影、紧凑内边距。
- 胶囊标签：灰底默认态、浅绿底 + 绿边选中态。
- 标题标签：黑色大字、选中下方浅绿短横。
- 底部固定按钮：白色渐变遮罩背景、绿色大圆角主按钮。
- 列表卡片：左图右文、标签行、用户和城市行、时间/报名数行。
- 表单块：白色圆角字段块、灰色输入区、标签在上、内容在下。
- 状态页：加载、空状态、错误状态统一为白卡或轻量居中提示。

### 图标策略

优先尝试接入 `@lucide/vue` 图标包。Lucide 图标默认使用 `currentColor`，适合通过样式变量统一控制黑色、灰色、绿色和红色状态。

如果 `uni-app` 的 H5 或微信小程序构建对 SVG 组件存在兼容问题，或引入图标包造成明显构建风险，则改为本地轻量线性图标方案，保持线性笔画、颜色和尺寸一致，不阻塞视觉还原。

## 核心页面设计

### 首页

首页对齐参考图 1 和 HTML 的 `HOME`：

- 顶部为自定义状态区下的搜索行，搜索框文案为“搜索摄影师 / 风格 / 城市”，右侧为彩色 AI 推荐圆形入口。
- 功能入口为 5 个线性图标宫格：约摄影师、约模特、妆造师、修图师、找同城。
- 主标签为推荐、同城、最新，选中项字体更大更黑，下方浅绿胶囊短横。
- 筛选入口位于标签右侧，点击打开底部筛选弹层。
- 快捷筛选胶囊横向滚动，选中态浅绿底、绿边、深绿文字。
- 约拍列表使用左图右文卡片，包含标题、目标标签、合作方式、实名认证、信用担保、头像昵称、城市、报名数和时间。
- 保留现有列表接口和分页逻辑，字段不足时使用默认图片或合理文案兜底。

### 发布约拍

发布页对齐参考图 2 和 HTML 的 `PUBLISH`：

- 顶部为返回按钮 + 居中标题“发布约拍”。
- 第一块为双分段大卡：我要约摄影师、我要约模特，选中态为浅绿背景、绿边和绿色图标。
- 表单字段包括标题、详细描述、合作方式、拍摄风格、城市、参考图/样片。
- 合作方式为单选胶囊；拍摄风格为多选胶囊。
- 图片上传为 3 列缩略图，上传入口为虚线圆角框。
- 底部为固定绿色“发布约拍”按钮，保留拍拍豆余额校验、图片处理和提交逻辑。
- 现有后端提交字段继续使用当前模型。标题字段在提交时并入现有 `describ` 描述内容；风格选项写入现有 `patstyle` 字段，不新增后端字段。

### 个人中心

个人中心对齐参考图 4 和 HTML 的 `MINE`：

- 顶部为菜单、扫码/功能入口、消息入口和红色未读角标。
- 用户信息包含头像、昵称、实名勾、关注数、粉丝数、约拍数、右侧箭头。
- 身份胶囊包含职业等级、信用分、实名状态。
- 会员卡为深色渐变，含“爱去拍 · 信用会员”和黄色开通按钮。
- 快捷入口为我的发布、我的约拍、我的收藏、钱包。
- 数据概览使用白色大卡，突出累计约拍、约拍币、被收藏、作品。
- 未登录状态使用同风格登录提示，不破坏整体布局。

## 其他页面延展

### 消息中心

消息中心使用白底标签切换和消息列表卡片。收到的、发出的保留现有数据源；消息项展示头像、昵称、内容摘要、城市和时间。未登录、加载和空状态使用统一状态组件。

### 约拍相关子页面

详情页按 HTML 还原大图顶部、白色详情 sheet、标签、作者卡、作品宫格、底部收藏/私信/立即约拍操作栏。

搜索、我的发布、我的申请、我的收藏复用首页列表卡片和筛选/状态样式，保证列表密度、卡片圆角和标签语言一致。

### 用户相关子页面

用户主页延展个人中心风格，展示头像资料、认证标签、数据摘要和作品/发布列表。

编辑资料、完善资料、实名认证、设置、意见反馈使用表单页模式：白色圆角字段块、灰底输入态、绿色主按钮。

钱包、充值、记录、账单使用数据卡模式：白色卡片、橙色金额强调、绿色主操作和列表行。

关于我们使用品牌卡片和信息列表；保持白底、绿主色和清爽排版。

### 内容页面

文章详情、用户协议、隐私政策、消息详情使用内容卡模式：标题、元信息、正文段落、浅灰背景和白色正文卡片。协议和隐私政策以可读性为优先，不做复杂装饰。

## 数据与错误处理

- 首页、消息、详情、我的发布/申请/收藏保留现有分页和下拉刷新。
- 发布页保留图片选择、图片转 `base64`、余额校验、提交中状态、失败提示和成功提示。
- 登录页保留微信小程序手机号授权逻辑；H5/App 不支持提示改为同风格白卡。
- 详情页保留收藏、报名、浏览量上报、登录拦截。
- 所有页面的加载、空状态、错误状态统一视觉，但不吞掉原有错误信息。
