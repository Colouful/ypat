# 导航与我的页面迁移缺口

## GAP-NAV-01：首页同城城市来源

当前新版没有经纬度反查城市接口。首页切到同城时会请求定位权限，但不能从坐标可靠得到城市名称，因此优先使用登录用户资料中的 `city` 字段；未登录或未设置城市时降级为全国。

需要后端或平台补齐：坐标反查城市能力，或统一的当前城市服务。

## GAP-MINE-01：好友邀请与奖励

旧版包含好友邀请页、分享路径和“赚拍拍豆”文案，但新版没有对应页面、邀请二维码、分享降级逻辑或奖励接口对接。

当前处理：保留入口，标记“功能完善中”，不增加虚假拍拍豆。

## GAP-MINE-02：信用担保缴纳

旧版 `creditflag === '1'` 时提示已信用保证，未担保时进入信用担保页面。新版缺少信用担保页面、保证金产品选择、支付创建、支付状态回查和重复缴纳校验页面逻辑。

当前处理：展示真实信用担保状态；未担保入口提示迁移中，不模拟支付。

## GAP-MINE-03：管理员角色与审核能力

旧版管理员来自 `adminList` 手机号命中后写入本地 `role=superAdmin`，不是后端权限字段。新版用户资料、Token 和当前接口未发现等价角色字段。

当前处理：普通用户不展示管理员模块；不建立空审核页。后续需要后端返回角色或权限列表，并在审核接口上做服务端权限校验。

## GAP-MINE-04：消息授权

旧版管理员“消息授权”调用订阅消息模板授权。新版缺少模板授权管理页面和管理员权限来源。

当前处理：不向普通用户展示，不用 Toast 假装完成。

## GAP-MINE-05：信息审核

旧版存在 `/pages/mine/infoaudit/index` 和 `/ypat/audit/list` 等审核能力。新版没有对应页面和直接路由权限保护。

当前处理：不迁移空页面；待管理员权限和审核页面设计补齐后再接入。
